Linux 2.2

The latest stable Linux kernel at the time of writing is v2.2.6, although many distributions still use 2.0.36. The list of changes is long - Open Resources magazine has a good explanation. Some of the highlights are:

Considerably improved performance on multiprocessor machines (with 2 to 16 CPUs), plus other performance tweaks such as improved caching and swapping routines

Specific optimisations for non-Intel processors, such as the Cyrix 6x86 and AMD K5 and K6

The EIDE driver supports UltraDMA and busmastering on most Pentium chipsets, and is removable for all-SCSI machines

Integration into the kernel of the functions of the previously separate kernel-module and automount daemons

However, the minimum memory footprint has risen from 4 to 6MB, and USB support wasn't finished in time to be included. Nor was DVD support, but most drives should work with the ordinary CD-ROM driver.

Some recent distributions (such as Red Hat 5.2 and SuSE 6.0) ship with the older kernel but claim to be "2.2 ready." In theory, this means that although they shipped before it was finished, the various supporting programs are versions that should function correctly with the new kernel. (There’s a full list at http://www.linuxhq.com/change22.html.) Still, upgrading to a 2.2.x kernel is a non-trivial exercise, and the improvements won't be relevant to everyone, especially if a machine is going to be performing a simple function such as a gateway.

Not only must many other components be updated, the boot and INIT structures have changed too - see the homepage for your distribution. Once an installation has been tweaked for 2.2, it often will no longer run 2.0.

As such, it’s not recommended to go to 2.2.x unless you are an expert - wait for the distributor to ship a version with the new kernel built-in. At the time of writing, SuSE 6.1, Caldera 2.2 and Slackware 4.0 are in final testing, and should be shipping by press time; a 2.2 version of Debian is already available for the intrepid.
