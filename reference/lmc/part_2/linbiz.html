Linux in Business

To get an idea of how people are using Linux in business, we asked some members of the CIX online system how they were using Linux.

"We’re prototyping a project using Oracle on Red Hat 5.1, running on a 64Mb AMD K6-2/266 with 3.5Gb of hard disk. Although we’re probably switching to Sun machines for the final product, we’re still going to be using PCs running Linux as our web servers. Installation was very straight-forward - I set the BIOS to boot from the CD-ROM and rebooted - and although I had minor problems with mount points and Xfree86, the instructions were clear enough. Putting the machine on the network wasn’t difficult using Samba, although the smb.conf file had to be edited manually to let the rest of the network see it."

Antony Shepherd, Derivative Trading Systems

"I work as a techie for a London web design crew. At the office we use Linux as the server/net access platform. We'd be a bit stuck without IP masquerading. At home (and I work most of the time from home) I use it as my main machine. Currently I'm working on a system that will be served from AIX - I'm using Linux to host the Web application server."

Jim Hague, Purple I

"We use Slackware Linux 3.5 for various internal engineering and development jobs, mostly as a source of documentation or an example of how things "should be done". There are a number of areas in which its flexibility really shines through, such as the ease with which an arbitrary program may be attached to activity on an incoming socket using the inetd daemon. Hardware includes a range of machines from a Pentium-II down to a 4MB 386/20 - it’s an indication of the maturity of the operating system that it runs without obvious race conditions on such a wide range of systems. We use Delphi under Win32 and a portable version of this would be extremely welcome, even if it used the GNU C compiler for code generation."

Mark Morgan Lloyd, KDG Telemetry

"We're currently using it on multiprocessor machines as an Internet-facing Web server, using Martin Poole's Secure Linux (LSSD). It provides a Geocites-style web presence and e-mail and DNS for the network. As this is to be sat on a co-hosted link in TeleCity, the system must be secure, able to stay up, and serve a large visitor base on a 24×7 basis. It’s also being used as an in-house development platform for some web-based work, for network monitoring using tools such as tcpdump and sniffit, and for reviewing security of servers and machines we set up for clients. We’re currently evaluating a test system using Oracle 8 as a replacement for a client's old SCO Unix system."

Stephen Kapp, DSL Software

"We've got 2 Linux boxes on a 'test' network within the Business Sales Group. One, which used to run Novell, is a fileserver, and the other runs Apache and MySQL, which we use for keeping annual leave records for people within the group. The most useful application is a quick statistics thing I wrote in Perl. We get a report from our IT Department every month telling us who sold what. This is imported in to another MySQL database and managers can drill down on statistics through a browser. It's been a big hit, and we’re now adding dial in access for the Sales & Marketing Director, who is on a different network."

Paul Crick, Jersey Telecom

"We use a pair of machines running Slackware with firewalled 2.0.3x kernels: a 32MB P133 Internet gateway (with IP masquerading), mostly for web browsing, and a 64MB P166 running sendmail and a message store for exchanging mail with the Internet and cc:Mail. Until recently, we were running on 486/66s with 32MB or less, but we upgraded to get better kernels and newer (and more secure) versions of sendmail etc. Compared to other UNIXes, Linux is a dream to install and maintain. Usually if there's a problem it's well-documented in a HOWTO, FAQ, or numerous web-sites and newsgroups."

Sean Pearce, Interlink Computer and Software Company
