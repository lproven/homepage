A brief guide to Unix

The Unix shell is powerful, difficult and different, and you'll need to use it a lot. To get you going, here's a short introduction to the essential commands and procudures.

Important differences

  1. Forward slashes - ‘/’ – separate directories, instead of DOS' backslashes - ‘\’.
  2. File and directory names are case-sensitive - the files readme, README and ReadMe are different and could exist in the same directory together.
  3. File extensions are used, but not universally, as native Unix files contain an internal "magic number" that tells the OS what type they are. You'll mainly find them on things like HTML files, or documents created by programs ported from other OSs.
  4. There are no drive letters. All filesystems, whether in different partitions or removable discs, appear as part of one big directory tree. There's an official layout for this, the Filesystem Hierarchy Standard. Some of the main directories are as follows:
Name  Function
/ Root directory
/bin, /sbin Binaries required for system startup and administration
/dev Devices (such as drives, partitions, I/O ports)
/etc, /var Configuration files
/mnt Mount point for temporary filesystems
/usr User files (such as system components, manuals, source code)
/lib, /usr/lib Shared libraries (similar to DLLs under Windows)
/tmp Temporary files
/opt Optional components (such as 3rd-party applications)
/home User home directories
/root Superuser's home directory

Broadly, you should install applications into /opt and keep data files in your home directory, and not change anything anywhere else. If you're upgrading a component, or have to edit a configuration file, always make a copy of the original first.

Essential commands

Terseness is part of the design ethos of Unix. Commands are abbreviated, and often only give any output if there's an error. Command switches are preceded by a dash, not a slash as in DOS. Like DOS, Ctrl-c stops a command running.

man <subject>
info <subject>

The most important command, and the equivalent of HELP under DOS. The standard Unix man command is often over-technical and rather uninformative, so you should use the GNU equivalent, info, which contains a lot of additional documentation. Start by studying info info. I. If info doesn't help, look for a HOWTO. You should find some preloaded in /usr/doc, or you can get the originals from Metalab or a local mirror. Other documentation can be found at the Linux Documentation Project and at the Linux Journal's Linux Resources page.

ls

List the files in a directory. Try adding -l (long listing), -a (shows hidden files, whose names begin with a full stop) and -R (lists subdirectories - note the capital!)

cd

Change directory. Works like DOS, except that cd on its own goes to the user's home directory.

pwd

Print working (current) directory.

cp

Copy files. Use -R (not -r) to copy subdirectories as well.

rm

Remove (delete) files. Take care, there's no recycle bin or undelete command! The command rm -r removes subdirectories and contents too, so use with extreme caution - for instance, typing rm -r * when logged in as root and in the root directory would wipe all files on all disks!

mv

Move or rename files and directories.

mkdir

Make directory.

rmdir <dirname>

Remove directory.

Finally, two complex commands often used to install software.

tar <options> <archive>

gunzip <archive>

Tar adds and removes files in archives, which usually have the extension .tar. It's similar to PKZIP under DOS, except that tar doesn't compress the data, so it's usually used with the GNU compressor gzip, yielding a `.tar.gz' extension. This combination is a common way to bundle up programs for distribution, and the resulting .tar.gz file is known as a "tarball".

To extract the files, first uncompress the tarball, then use tar to extract the contents. First, make a copy to work on. Then, unzip it:

# gunzip <tarballname>.tar.gz

The gunzip command will happily uncompress its own .gz files, plus those from PKZip, WinZip and the Unix pack and compress commands. Note that it removes the original archive after uncompressing it.

Then, extract the files from the archive:

# tar -xvf <tarballname>.tar

This unpacks all the files and directories in the archive into the current directory, reproducing the directory structure and listing them as it goes.

mount <device> <directory>

Insert a filesystem (say, a partition, a CD or a floppy disk) into the directory tree. The contents of the filesystem replace the contents (if any) of the directory.

logout

(Or Ctrl-d) terminates the current shell.
